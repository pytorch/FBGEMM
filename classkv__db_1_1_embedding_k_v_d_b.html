Detailed Description

Currently it is used by TBE to offload Key(Embedding Index) Value(Embeddings) to DRAM, SSD or remote storage, to provide better scalability without blowing up HBM resources

◆ get()

void get ( const at::Tensor & indices,
const at::Tensor & weights,
const at::Tensor & count,
int64_t sleep_ms = 0 )

Find non-negative embedding indices in <indices> and fill up the related embeddings into <weights> for the first <count> elements in the tensor. It contains 3 steps:

  • L2 cache lookup
  • filling weights concurrently from L2 cache(cache hit) and storage tier(Rocksdb/PS) on cache miss
  • for cache misses, copy 3 tensors and inject them into the background working item queue to be picked up for eviction/promotion handling between L2 cache and storage tier
Parameters
indicesThe 1D embedding index tensor, should skip on negative value
weightsThe 2D tensor that each row(embeddings) is paired up with relative element in <indices>
countA single element tensor that contains the number of indices to be processed
sleep_msthis is used to specifically sleep in get function, this is needed to reproduce synchronization situation deterministicly, in prod case this will be 0 for sure
Returns
None
Note
weights will be updated from either L2 cache or storage tier
